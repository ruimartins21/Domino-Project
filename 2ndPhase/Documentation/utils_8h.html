§ ALLSEQUENCES

typedef struct allSequences ALLSEQUENCES

Structure that gathers all the sequences generated, saving the number of sequences and the first sequence of the linked list

§ BLOCK

typedef struct block BLOCK

The block structure is where all the information about a block is stored, both sides, if it's available for use or not the pointer for the next block, and a pointer to the previous one since it's a doubly linked list

§ GAME

typedef struct game GAME

The game structure is where all the blocks are initially stored before any hands are created and it's used to transfer the blocks for the hands and for swapping from other blocks existing on any hand this way there's no chance of repeating blocks

§ HAND

typedef struct hand HAND

This structure stores all the blocks within a hand, it has a pointer to the first block since it's a linked list and a pointer to the next hand

§ HANDS

typedef struct hands HANDS

Structure that gathers all the hands used in a game, it stores each hand size (that is the same for all) it stores the number of hands existing and the pointer to the first hand since it's a linked list

§ ID

typedef struct id ID

Structure for utility purpose only, it's a linked list of ids of sequences

§ IDS

typedef struct ids IDS

Structure for utility purpose only, it's used to store the Ids of the sequences that match a pattern, to limit user choice when choosing a sequence to replace

§ SEQUENCE

typedef struct sequence SEQUENCE

Structure that holds all the blocks used on a certain sequence generated, the sequence size and the next sequence, since it's a linked list

§ STRINGSEQ

typedef struct stringseq STRINGSEQ

Structure that does the same job as the SEQUENCE one but instead of a linked list of blocks and its side as integers this one saves the hole sequence as a string for searching and replacing purposes
